# AGENTS.md

## Cursor Cloud specific instructions

### Overview

This is the **ConductorQuantum Python SDK** — a client library for the Conductor Quantum API. It has two product lines:
- **Control** — ML-based analysis models for quantum device characterization (upload numpy arrays, get results)
- **Coda** — Quantum circuit tools, QPU access, and AI agent chat via SSE streaming

The SDK is partially auto-generated by [Fern](https://buildwithfern.com); hand-written code lives under `.fernignore`.

### Development commands

All commands are run via `poetry run` from the repo root. The CI check script `scripts/local_check_ci.sh` is the canonical reference.

| Task | Command |
|---|---|
| Lint | `poetry run ruff check src/ tests/` |
| Format check | `poetry run ruff format --check src/ tests/` |
| Type check | `poetry run mypy .` |
| Unit tests | `poetry run pytest tests/ --ignore=tests/custom/test_models_integration.py --ignore=tests/custom/test_coda_integration.py -v` |
| Full local CI | `bash scripts/local_check_ci.sh` |
| Integration tests | `bash scripts/local_check_ci.sh --integration` (requires `CONDUCTOR_QUANTUM_API_KEY`) |

### Gotchas

- **Pre-existing lint issues**: The repo has import-sorting violations (`I001`) in several test files. These are in auto-generated code and should not be fixed without coordinating with Fern regeneration.
- **Ruff format**: Two test files (`test_coda_integration.py`, `test_models_integration.py`) have pre-existing format differences.
- **No local services**: This is a pure client SDK. There are no databases, Docker containers, or backend servers to run. All unit tests use `httpx.MockTransport` and run fully offline.
- **Integration tests require API keys**: Set `CONDUCTOR_QUANTUM_API_KEY` (or separate `CONTROL_API_TOKEN` / `CODA_API_TOKEN`) environment variables. Coda tokens must start with `coda_`.
- **Poetry must be on PATH**: After pip-installing Poetry, ensure `$HOME/.local/bin` is on `PATH`.
